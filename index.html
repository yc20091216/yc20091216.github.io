1. 说说库和框架的区别？
  • 库：将代码集合成一个产品，提供给开发者调用。面向对象的代码组织而成的叫类库，面向过程组织而成的叫函数库。
  • 框架：为解决一个(一类)问题而开发的产品，框架用户一般只需要使用框架提供的类或函数，即可实现全部功能。
  • 区别：库提供更灵活的选择，需要自己组织。框架更具有针对性，提供一整套完整的代码。

1. Jquery 中， $(document).ready()是什么意思？和window.onload 的区别？ 还有其他什么写法或者替代方法？
  • $(document).ready():当DOM加载完成后执行该函数；
  • window.onload:当页面加载完成后执行该函数(页面内容、图片、外部资源)。
  • 两者区别：
    • 执行的事件不同：$(document).ready()当DOM构建完成后就执行,而window.onload需要等页面内所有资源加载完成后。
    • 编写个数不同：window.onload只能执行一次，执行最后一个window.onload。$(document).ready()可以执行多个。
    • window.onload没有其他写法,$(document).ready(handler)还有$(handler)、$().ready(handler)。
2.$node.html()和$node.text()的区别?
  • $node.html():返回该节点下的html内容(包括html标签)。
  • $node.text()：返回该节点下的文本内容(不包括html标签)。
3. $.extend 的作用和用法?
  • 语法: jQuery.extend([deep,] target [, object1 ] [, objectN ] );
  • 作用：
    • 当我们提供两个或多个对象给$.extend()，对象的所有属性都添加到目标对象（target参数）。
    • 如果只有一个参数提供给$.extend()，这意味着目标参数被省略。在这种情况下，jQuery对象本身被默认为目标对象。这样，我们可以在jQuery的命名空间下添加新的功能。这对于插件开发者希望向 jQuery 中添加新函数时是很有用的。
    • 如果想保留原对象:var object = $.extend({}, object1, object2);
  • 默认情况下，通过$.extend()合并操作不是递归的：
4. JQuery 的链式调用是什么？
  • jQuery链式调用：在对象上一次性调动多个方法
    • $(this).addClass("active").siblings().removeClass("active")
  • 因为大部分对象方法的最后是return this，所以有了链式调用，简易代码。
5. JQuery ajax 中缓存怎样控制?
  • 缓存：当向服务器请求数据时，get类型会产生缓存，而post不会产生缓存；如果下次请求相同的话，浏览器会从本地缓存读取数据而不会从服务器获取数据，这将可能导致数据不同步造成差异。
  • 利用cache参数来控制缓存。
    • cache (默认: true, dataType为”script”和”jsonp”时默认为false)，如果设置为 false ，浏览器将不缓存此页面。
6. jquery 中 data 函数的作用？
  • 作用：data方法允许在DOM元素上绑定任意类型的数据，避免了循环引用的内存泄漏风险。
  • 语法：$(“selector”).data(key, value)。

    • 根据用户输入的域名解析为对应ip地址。
    • 尝试连接对应IP地址，发送打包请求php文件。

    • php应用服务器解析后翻译成html文件返回给web服务器，由web服务器将翻译后的html文件返回给客户端浏览器。
    • 浏览器接受服务器返回html文件内容并解析渲染呈现在浏览器窗口。
  • 用户端访问服务器端的数据库：
    • 根据用户输入的域名解析为对应ip地址。
    • 尝试连接对应IP地址，发送打包请求php文件。
    • 由web服务器委托php应用服务器处理接受的打包请求。
    • 在php中通过对数据库的连接代码来连接对应数据库，并且由php程序执行标准SQL查询语句获取数据返回给php应用服务器，php应用服务器解析后翻译成html文件返回给web服务器，由web服务器将翻译后的html文件返回给客户端浏览器。

7. attachEvent与addEventListener的区别？

  • attachEvent:是IE的事件监听器。接受两个参数，第一个参数是事件类型，格式为”on+type”(onclick),第二个参数是事件处理程序函数。事件处理程序只能发生在冒泡阶段。
  • addEventListener:是W3C标准的事件监听器。接受三个参数，第一个参数时事件类型，格式为”type”(click),第二个参数是事件处理程序函数，第三个参数是布尔值,默认为false(冒泡事件),true为捕获阶段事件。

8. 解释IE事件冒泡和DOM2事件传播机制？

  • IE事件冒泡：事件开始时由最具体的元素接收，然后逐级向上传播到较为不具体的元素。
  • DOM2事件传播机制：DOM2事件传播机制属于DOM事件流(DOM2级事件规定事件流包括三个阶段，事件捕获阶段，处于目标阶段，事件冒泡阶段，首先发生的是事件捕获，为截取事件提供机会，然后是实际目标接收事件，最后是冒泡阶段)，并且第三个参数可以设置事件触发时是捕获阶段或者冒泡阶段。

1. \d，\w,\s,[a-zA-Z0-9],\b,.,*,+,?,x{3},^$分别是什么?

  • \d:匹配 0-9之间的数字。
  • \w:匹配所有字母，数字和下划线。
  • \s:匹配所有空白，包括空格，换行、tab缩进等。
  • [a-zA-Z0-9]:匹配字母和数字。
  • \b(元字符):边界 boundary，\b并不匹配任何字符，它只匹配一个位置，代表着单词的开头或结尾，也就是单词的分界处。
  • .(元字符):匹配除了换行符以外的任意字符。
  • (限定符):匹配前面的字符连续重复任意次。
  • +(限定符):匹配+前面的字符连续重复一次或更多次。
  • ？(限定符):匹配?前面的字符出现1次或者0次。
  • x{3}:匹配至少3个连续的xxx。
  • ^(元字符):匹配字符串的开始。
  • $(元字符):匹配字符串的结束。

2. 贪婪模式和非贪婪模式指什么?

  • 贪婪模式：当正则表达式中包含能接受重复的限定符时，通常的行为是（在使整个表达式能得到匹配的前提下）匹配尽可能多的字符。

  • 懒惰模式：（在使整个表达式能得到匹配的前提下）匹配尽可能少的字符。

  • 闭包是指有权访问另一个函数作用域中的变量的函数。

  • 这些函数可以“记忆”它被创建时候的环境。

    function adder(x){
  return function(y){
    return x + y ;
  }
}
var a = adder(5);
var b = adder(6);

console.log(a(5));  // 10;
console.log(b(5));  // 11;

    － 声明了a,b;它们共享了函数adder定义，却保存了不同环境，这时的a,b是两个闭包。

2.setTimeout 0 有什么作用?

  • setTimeout(“code”,millisec):执行代码必须用字符串形式，millise设置在多少毫秒之后执行这段代码。
  • setTimeout(fn,millisec):fn为函数名，millise设置在多少毫秒之后执行这个函数。
  • 如果省略第二个参数，该参数默认为0.
  • setTimeout():返回一个数字编号，该编码作为参数用来取消这次延迟调用。

  • 例子，下面代码效果一致。

    setTimeout("console.log('延迟了1000毫秒')",1000);
setTimeout(fn,1000);
function fn(){
  console.log("延迟了1000毫秒")；
}
  • setTimeout(function,0):将此函数放在JS代码末端立即执行。

  • 例子：

    setTimeout('console.log(1)',1000); //右边为执行顺序:  2
console.log(2);                    //               3
console.log(3);                    //               4
setTimeout('console.log(4)',0);    //               1

基础类型有哪些？复杂类型有哪些？有什么特征？

  • 基本数据类型：Undefined、Null、Boolean、Number、String(简单的数据段)。
    • 基本数据类型是按值访问的，因为可以操作保存在变量中的实际的值。
  • 复杂类型(引用类型)：Object(可能由多个值构成的对象)。
    • 复杂类型的值是保存在内存中的对象。

  • 特征：

    • 只能给引用类型值动态的添加属性，以便将来使用。
      var student = {};
      student.name = “Jhon”;
      console.log(student.name); // Jhon
      var name = “Jhon”
      name.age = 18;
      console.log(name.age) // Undefined;

    • 复制变量值：

      • 如果从一个变量向另一个变量复制基本类型的值，会在变量对象上创建一个新值，然后把该值复制到新变量分配的位置上，这两个变量可以参与任何操作而不会互相影响。
        var count1 = 18;
        var count2 = count1;
        count1 = 20;
        console.log(count1); // 20
        console.log(count2); // 18
      • 当从一个变量向另一个变量复制引用类型的值时，这个值得副本实际上是一个指针，指向同一个对象，因此改变一个变量就会影响到另一个变量。
        var count1 = {first:18};
        var count2 = count1;
        count1.first = 30;
        console.log(count1.first); // 30
        console.log(count2.first); // 30

数组方法里push、pop、shift、unshift、join、split分别是什么作用?

  • push()方法: 在数组的最后添加一个或者多个元素,并且返回当前的length。
    var a = [1,2,3,4];
a.push(5);
console.log(a);      // a = [1,2,3,4,5]
console.log(a.length);  // 5
a.push(6,7,8);
console.log(a);    // a = [1,2,3,4,5,6,7,8]
console.log(a.length);  // 8
  • pop()方法:用于删除并返回数组的最后一个元素。
    var a = [1,2,3,4];
a.pop();            //返回删除的值 4；
console.log(a);     //  a = [1,2,3]
console.log(a.length);  //  3
  • shift()方法：用于删除并返回数组的最前面一个元素。
    var a = [1,2,3,4];
a.shift();            //返回删除的值 1；
console.log(a);     //  a = [2,3,4]
console.log(a.length);  //  3
  • unshift()方法：在数组的最前面添加一个或者多个元素，并返回当前的length。
    var a = [1,2,3,4];
a.unshift(0);
console.log(a);      // a = [0,1,2,3,4]
console.log(a.length);  // 5
a.unshift(-3,-2,-1);
console.log(a);    // a = [-3,-2,-1,0,1,2,3,4]
console.log(a.length);  // 8
  • jion()方法：把数组元素（对象调用其toString()方法）使用参数作为连接符连接成一字符串，不会修改原数组内容。
    var a = [1,2,3,4];
console.log(a.join());//不填写参数时默认逗号作为连接符 "1,2,3,4"
console.log(a.join("")); //参数为空字符串时输出  "1234"
console.log(a.join("-"));  // "1-2-3-4"
console.log(typeof a);  //  object
console.log(a);       // [1,2,3,4]

  • split()方法：把一个字符串分割成字符串数组。默认不输入参数时字符串整体作为一项数组。有两个参数，第一个参数作为分隔符来分隔元素，第二个是分割出来的数组项个数。

    var a = "1a2b3c4d";
console.log(a.split());        // ["1a2b3c4d"]
console.log(a.split("b"));     //["1a2", "3c4d5e"]
console.log(a.split(""));      //["1", "a", "2", "b", "3", "c", "4", "d", "5", "e"]
console.log(a.split("",3));    //["1","a","2"]   
console.log(typeof a);        //"string"

  • reverse()方法：颠倒数组中元素的顺序，修改并返回改变后的数组。

    var a = [1,2,3,4,5];
a.reverse();
console.log(a);   //  [5,4,3,2,1];

  • slice()方法：提取原数组的一部分，返回一个新数组，原数组不变。

    var a = [1,2,3,4,5];
console.log(a.slice());  // [1,2,3,4,5]
console.log(a.slice(3));  //[4,5]
console.log(a.slice(0,3)); //[1,2,3]
console.log(a.slice(-3,-1));  //[3,4] 参数为负数表示倒数位置。
console.log(a);   //[1,2,3,4,5]  原数组不变。

  • map()方法：对数组的所有成员依次调用一个函数，根据函数结果返回一个新数组,原数组不变。

    var a = [1,2,3,4,5];
console.log(a.map(function ( n ){
               return n + 1;
}));    //[2,3,4,5,6]
 console.log(a)；  // [1,2,3,4,5]

函数声明和函数表达式有什么区别？

  • 函数声明：
    function FunName(){}

  • 函数表达式：var FunName = function (){}

  • 区别：使用函数声明会使这个函数声明前置（在指定作用域中），可以在定义前或后被调用执行。
    使用函数表达式只提升变量（函数表达式不会前置），等到定义这个函数并赋值给变量后才能被调用或执行。

什么是函数的作用域链 ？

  • 函数作用域：作用域就是变量与函数的可访问范围，即作用域控制着变量与函数的可见性和生命周期。
    • 全局变量：可以在脚本中的任何位置被引用。
    • 局部变量：只存在于声明它的函数内部，在那个函数的外部无法引用它。

  • 看看下面的几种情况：

    var a = 1;  //全局变量

function FunName(){

   console.log(example);
   result = "你好";   //没有使用var声明，result为全局变量。
   return result;

}    
   var result = "Hi"    //此声明已被覆盖。      
     FunName();
      //  "你好"    
     console.log(example);   //局部变量只有该函数内部能引用。

  • 作用域链：内部上下文所有变量对象（包括父变量对象）的列表，用来变量查询。在代码执行的过程中，所用到的变量会在当前作用域中进行寻找，如果找不到，就会往沿着作用域链向上一级进行寻找，一直到全局作用域为止，如果找到便会停止（而不理会上一级是否有同名的变量），如果找不到，就会报错。
