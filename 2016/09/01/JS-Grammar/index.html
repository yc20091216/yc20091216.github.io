JS Grammar

发表于 2016-09-01

CSS和JS在网页中的放置顺序是怎样的？

  • CSS应放置在：<head>~<style></style>~</head>
  • js应放置在：HTML文档的最后，</body>标签之前
    各个浏览器渲染机制可能略有偏差，但为了防止出现白屏或者FOUC等情况，JS放置位置还有可能导致网页加载出错，所以应当将JS和CSS分别放置在上面规定的位置。

解释白屏和FOUC

  • 白屏：浏览器的页面在切换时，由于其页面加载机制，在跳转到下一个页面时，先要请求联网、载入页面代码、构建dom、渲染，最后才显示出来。如果样式文件较大，又多，把样式放底部，结合网速和计算机性能，对于IE浏览器，在某些场景下会出现白屏，而不是内容逐步展现。@import标签即使放在头部也有可能出现白屏。

  • FOUC：将样式放在底部或引入@import，IE 会先加载整个 HTML 文档的 DOM ，然后再去导入外部的 CSS 文件，因此，在页面 DOM 加载完成到 CSS 导入完成中间 会有一段时间页面上的内容是没有样式的，时间长短跟网速和计算机性能有关。

  • JavaScript加载的同时会阻塞后面内容的呈现和阻塞其他组件的下载，或者将JS文件引入放顶部也同样会导致白屏或者FOUC出现的可能。

async和defer的作用是什么？有什么区别

首先脚本需要可用，它们仅适用于外部脚本。

  • async：加载和渲染后续文档元素的过程将和 script.js 的加载与执行并行进行（当页面继续进行解析时，脚本将被执行）。

  • defer：加载后续文档元素的过程将和 script.js 的加载并行进行（异步），但 script.js 的执行要在所有元素解析完成之后，DOMContentLoaded 事件触发之前完成。

  • 区别：async：不保证加载JS后执行的顺序。defer：脚本延迟到文档解析和显示后执行，有顺序。

简述网页的渲染机制

  • 解析HTML标签，构建DOM树。
  • 解析CSS标签，构建CSSOM树。
  • 通过DOM Tree 和 CSS Rule Tree 来构造 Rendering Tree。
  • 在渲染树的基础上，计算每个节点的几何机构。
  • 通过调用操作系统Native GUI的API绘制。

JavaScript 定义了几种数据类型? 哪些是简单类型?哪些是复杂类型?

  • 数值(number)：整数、小数（浮点数）。
  • 字符串(string)：由零个或多个字符构成，包括（但不限于）字母、数字、标点符号和空格。字符串必须包括在单、双引号里。
  • 布尔值：只有两个可选值，true或者false。
  • null：表示空缺，此处应该有一个值，但目前为空。
  • undefined：表示“未定义”或不存在。
  • 对象：各种值组成的集合（复杂类型）。

NaN、undefined、null分别代表什么?

NaN：（Not a Number），非数值，不等于任何值包括它本身。
undefined：”未定义”或不存在。
null：表示空值，即该处的值现在为空。比如，调用函数时，不需要传入某个参数，这时就可以传入null。

typeof和instanceof的作用和区别?

  • typeof：typeof运算符可以返回一个值的数据类型。
    var a=1; b="b"; c=true typeof a; //"number" typeof b; //"string" typeof c; //"boolean"

  • instanceof：主要的目的是检测引用类型,判断对象是Array，object,function。
    var array=new Array(); console.log(array instanceof Array); // true

  • 区别：typeof和instanceof的目的都是检测变量的类型,两个的区别在于typeof一般是检测的是基本数据类型,instanceof主要检测的是引用类型!
