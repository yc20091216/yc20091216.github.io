函数声明和函数表达式有什么区别？

  • 函数声明：
    function FunName(){}

  • 函数表达式：var FunName = function (){}

  • 区别：使用函数声明会使这个函数声明前置（在指定作用域中），可以在定义前或后被调用执行。
    使用函数表达式只提升变量（函数表达式不会前置），等到定义这个函数并赋值给变量后才能被调用或执行。

  • 变量声明前置：只提升变量声明（到指定作用域顶部），并不会提升赋值操作。
  • 函数声明前置：使用函数声明会提升到指定作用域顶部，使用表达式只提升该函数所赋值的变量。

立即执行函数表达式是什么？有什么作用 ？

  • (Immediately-Invoked Function Expression)简称IIFE：

    • !function (){}();
    • (function(){});

  • 防止变量产生全局污染。（当同时调用多个JS的情况时，很容易造成因为变量名一致或函数名一致而覆盖其他库的变量。）

什么是函数的作用域链 ？

  • 函数作用域：作用域就是变量与函数的可访问范围，即作用域控制着变量与函数的可见性和生命周期。
    • 全局变量：可以在脚本中的任何位置被引用。
    • 局部变量：只存在于声明它的函数内部，在那个函数的外部无法引用它。

  • 看看下面的几种情况：

    var a = 1;  //全局变量

function FunName(){
   var example = "chestnut" ;  //使用var声明,example为局部变量。
   console.log(example);
   result = "你好";   //没有使用var声明，result为全局变量。
   return result;

}    
   var result = "Hi"    //此声明已被覆盖。      
     FunName();
      //  "你好"    
     console.log(example);   //局部变量只有该函数内部能引用。

  • 作用域链：内部上下文所有变量对象（包括父变量对象）的列表，用来变量查询。在代码执行的过程中，所用到的变量会在当前作用域中进行寻找，如果找不到，就会往沿着作用域链向上一级进行寻找，一直到全局作用域为止，如果找到便会停止（而不理会上一级是否有同名的变量），如果找不到，就会报错。
    参考JavaScript 开发进阶：理解 JavaScript 作用域和作用域链
